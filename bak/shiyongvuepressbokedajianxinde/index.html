使用vuepress博客搭建心得

renxian2025/08/07vue

一、前言

以前自己写博客网站（后台写文章发布那种），买服务器搭建，一整套下来费时费力还费钱（虽然不爱写，只是觉得有必要有一个自己的网站😂）。

后来服务器到期了，然后不知时候不小心把代码也删了（估计清项目文件的时候吧，也没在Github上建仓库😭），就干脆不自己写了，转向静态网页生成加Github Page，现在用了一段时间了，真香。

这里记录一下心得吧，还是花了一些时间去研究这个框架的，希望能帮助一些想搭博客少走一些弯路。

二、搭建过程

我这里选的主题为reco2.0，简洁还好看，深得我心。官网：vuepress-reco-themeopen in new window。

1. 创建项目

reco官网提供了一个脚手架，可以快速创建项目，推荐使用，比自己配依赖和建目录快多了。

我使用的npm：

# 初始化，并选择 2.x
npm install @vuepress-reco/theme-cli@1.0.7 -g
theme-cli init

创建好后进入项目录运行包管理器的install命令安装依赖便可，运行结果：

2. 项目结构

reco主题和原主题结构基本保持一致，主要增加了博客文章的目录结构。

这里列出一些重要的目录和文件：

- .vuepress  :资源和配置目录
  - public  :静态资源目录
  - styles  :样式目录
  - config.ts  :主题配置文件
- blogs  :博客文章目录
- docs  :普通文档目录
- series  :系列文章目录
- README.md  :首页内容

主要修改的便是config.ts和README.md文件。

3. 配置主题

主题配置其实没有太多要说了，很简单的一部分，按照自己需求配置即可，这里放一下我的配置吧，主要说一下怎么去看可以用的配置。

主要看defineUserConfig和recoTheme参数。

defineUserConfig参考vuepress配置open in new window。vuepress配置文档还是很完善的，基本可以根据文档按需配置。如果不打算使用reco主题，可以直接使用vuepress的默认主题open in new window。

recoTheme参数参考reco主题配置open in new window。

import {defineUserConfig} from "vuepress";
import recoTheme from "vuepress-theme-reco";
import {viteBundler} from '@vuepress/bundler-vite'

export default defineUserConfig({
    title: "散客", // 网站标题
    description: "thank", // 网站描述
    lang: "zh-CN", // 网站语言
    head: [
        ["link", {rel: "icon", href: "/logo.svg"}],
    ],
    bundler: viteBundler(),
    theme: recoTheme({
        logo: "/logo.svg",
        author: "renxian",
        authorAvatar: "/avatar.jpg",
        docsRepo: "https://github.com/renxian-xx/renxian-xx.github.io",
        docsBranch: "main",
        lastUpdatedText: "上次更新于",
        navbar: [
            {text: "首页", link: "/"},
            {text: "分类", link: "/categories/huanshijiemozukaifa/1.html"},
            {text: "标签", link: "/tags/Mod/1.html"},
            {text: "时间线", link: "/timeline.html"},
        ],
        primaryColor: "#97caf9",
        editLink: false,
        categoriesText: "分类",
        tagsText: "标签"
    }),
});
