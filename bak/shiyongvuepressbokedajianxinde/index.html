使用vuepress博客搭建心得

renxian2025/08/07vue

一、前言

以前自己写博客网站（后台写文章发布那种），买服务器搭建，一整套下来费时费力还费钱（虽然不爱写，只是觉得有必要有一个自己的网站😂）。

后来服务器到期了，然后不知时候不小心把代码也删了（估计清项目文件的时候吧，也没在Github上建仓库😭），就干脆不自己写了，转向静态网页生成加Github Page，现在用了一段时间了，真香。

这里记录一下心得吧，还是花了一些时间去研究这个框架的，希望能帮助一些想搭博客少走一些弯路。

二、搭建过程

我这里选的主题为reco2.0，简洁还好看，深得我心。官网：vuepress-reco-themeopen in new window。

1. 创建项目

reco官网提供了一个脚手架，可以快速创建项目，推荐使用，比自己配依赖和建目录快多了。

我使用的npm：

# 初始化，并选择 2.x
npm install @vuepress-reco/theme-cli@1.0.7 -g
theme-cli init

创建好后进入项目录运行包管理器的install命令安装依赖便可，运行结果：

2. 项目结构

reco主题和原主题结构基本保持一致，主要增加了博客文章的目录结构。

这里列出一些重要的目录和文件：

- .vuepress  :资源和配置目录
  - public  :静态资源目录
  - styles  :样式目录
  - config.ts  :主题配置文件
- blogs  :博客文章目录
- docs  :普通文档目录
- series  :系列文章目录
- README.md  :首页内容

主要修改的便是config.ts和README.md文件。

3. 配置主题

主要看defineUserConfig和recoTheme参数。

defineUserConfig参考vuepress配置open in new window。vuepress配置文档还是很完善的，基本可以根据文档按需配置。

recoTheme参数参考reco主题配置open in new window ，reco主题的文档比较零散，可以参见recoTheme的d.ts文件，相比官方文档看着要舒服很多。

主题配置其实没有太多要说了，很简单的一部分，按照自己需求配置即可，这里放一下我的配置吧，具体效果看博客首页。

import {defineUserConfig} from "vuepress";
import recoTheme from "vuepress-theme-reco";
import {viteBundler} from '@vuepress/bundler-vite'

export default defineUserConfig({
    title: "散客", // 网站标题
    description: "thank", // 网站描述
    lang: "zh-CN", // 网站语言，html标签上的lang属性
    head: [ // 网站头部标签，配置meta、link等
        ["link", {rel: "icon", href: "/logo.svg"}],
    ],
    bundler: viteBundler(), // 项目使用的打包器，这里使用vite打包器，vuepress默认的打包器也是vite
    theme: recoTheme({ // 配置主题，使用reco主题
        logo: "/logo.svg", // 网站logo，位于public目录下，可以通过vuepress的public属性更改
        author: "renxian", // 作者名称
        authorAvatar: "/avatar.jpg", // 作者头像，位于public目录下
        navbar: [ // 顶部网站导航栏配置，有一些内置的页面具体看reco官网
            {text: "首页", link: "/"},
            {text: "分类", link: "/categories/huanshijiemozukaifa/1.html"}, // 路径规则：/categories/{分类名字（中文的就是拼音）}/{页码}.html
            {text: "标签", link: "/tags/Mod/1.html"}, // 路径规则：/categories/{标签名字（中文的就是拼音）}/{页码}.html
            {text: "时间线", link: "/timeline.html"},
        ],
        primaryColor: "#97caf9", // 主题颜色
        docsRepo: "https://github.com/renxian-xx/renxian-xx.github.io", // 文章仓库，这个是配合编辑文章功能（editLink参数）使用的，会跳转到仓库文章位置
        docsBranch: "main", // 文章仓库分支也是配合编辑文章功能使用的
        editLink: false, // 是否启用编辑文章功能，启用后会在文章底部显示编辑链接，点击后会跳转到仓库对应文章位置
        // 下面这些就是网站的文本，主要是做国际化用的
        lastUpdatedText: "上次更新于", // 文章底部的更新时间文本 
        categoriesText: "分类", // 首页的分类文本
        tagsText: "标签" // 首页的标签文本
    }),
});

4. 首页配置frontmatter

首页的配置主要是frontmatter，相比之下reco的首页文档要完善一些：reco主题首页frontmatteropen in new window ，根据文档按需配置即可。

我的配置如下，我喜欢简洁，所以没有配置太多模块。

---
home: true
modules:
  - Blog
  - Footer
---

5. 编写文章

1) 博客文章

博客文章放在blogs目录下，使用markdown编写，文件名任取，属于frontmatter配置文章属性。

vuepress文章frontmatteropen in new window

reco主题特有frontmatteropen in new window

案例：

---
title: java-使用自定义注解代替@RequestBody实现根据参数名注入json数据
date: 2025/08/06 00:00:00
tags:
  - Java
---
